能够辨别程序中异常和错误的区别
	异常： Exception， 可以挽回的问题。
	错误： Error， 不可以挽回的问题。
说出异常的分类
	运行时异常。
	编译时异常。
说出虚拟机处理异常的方式
	如果程序中出现了异常，并且没有使用try...catch捕获处理，那么这个异常会往外抛。
列举出常见的三个运行期异常
	空指针， 索引越界， 除0异常  10/0
能够使用try...catch关键字处理异常
	try {
	
	} catch(要捕获的异常) {
	
	}
能够使用throws关键字处理异常
	如果在方法内抛出了编译时异常，则必须要使用throws进行声明。
	调用了一个throws声明了异常的方法，则调用者也要进行声明。
能够自定义异常类
	认贼作父
能够处理自定义异常类
	在自定义异常类中提供构造方法，传递异常信息。
	并且在抛出异常类对象的时候给出异常信息。
说出进程的概念
	正在运行的程序
说出线程的概念
	进程中的执行单元
能够理解并发与并行的区别
	并发同一个时间段，有多个线程同时执行。
	并行同一个时间点，有多个线程同时执行。
能够开启新线程
	1. 定义类继承Thread
	2. 重写run方法。
	3. 在测试类中创建Thread子类对象
	4. 调用start方法，让这个线程运行。
